# 现代法教义学风格资源

> 本文件是词汇与句法资源池，不是独立分析工作流。是否可以增加理论、检验要件或结论，始终由 `task_mode` 控制。

<!-- chunk: r08.h001 -->
## 一、风格特征

- 概念精确、层次清楚；
- 偏好名词化、体系定位、条件限定与类型区分；
- 句法可以复杂，但结论强度必须与来源一致；
- 外语术语是可选括注，不是权威装饰；
- 不默认生成“通说—少数说—本文见解”。

<!-- chunk: r08.h002 -->
## 二、rewrite 可用资源

仅在与原意等值时使用：

- 规范评价、关系结构、意思形成、效力范围；
- 行为形态、结果状态、因果关联、条件门槛；
- 权利主体、义务主体、相对人、处分主体；
- 事实层面、规范层面、时间维度、关系维度；
- 构成要件式描述，但不得自动进入完整构成要件检验。

常用句法：

```text
从【层面】观察，……
该问题并非仅在于……，更在于……
【事实】与【效果】分属不同层次。
只有在【明示条件】具备时，方能……
```

## 三、expand 可用资源

可在预算内加入：

- 信赖保护；
- 风险分配；
- 意思自治；
- 法益保护；
- 制度功能；
- 权利边界；
- 程序参与；
- 比例性或利益衡平；
- 事实状态与规范效果的区分；
- 形式判断与实质判断的区分。

使用方式应为观察框架或概念区分，不得写成既定适用结论。

## 四、analyze 专属资源

以下结构仅在 `analyze` 开放：

- 三阶层完整检验；
- 请求权基础与抗辩结构；
- 客观归责完整检验；
- 学说对立、通说与少数说；
- 具体法条、判例、学者与比较法依据；
- 涵摄后形成的责任或效力结论。

即使处于 analyze，也必须有事实与权威基础。

## 六、避免事项

- 不把任何刑法句子自动写成三阶层；
- 不把任何民法句子自动写成请求权基础；
- 不把“可能”改成“成立”；
- 不将“行为导致结果”自动升级为客观归责成立；
- 不为了教义感虚构争议学说；
- 不以固定外语占比堆叠术语；
- 不执行本文件之外的独立生成流程。

## 七、德日理论翻译腔

特征：定语长链、框形结构、抽象主语、动词后置感。可用于 `doctrinal_dense`；日常输入不得整段移植。

```text
标准形式：对【对象】之【名词化动作】，系【判断】。
高级形式：就【事项】而言，作为【上位范畴】之【概念】，其【属性或功能】，应在【体系位置】之脉络下，依【标准】加以确定。
框形结构：在【前提或条件】之下，【主体】对【对象】所为之【行为】，具有【效果或意义】。
```

使用规则：框形结构每段至多一次；长定语链不得超过三层嵌套；翻译腔不得牺牲可回译性——拆回短句后命题必须不变。

## 八、概念递归、层级分类与概念套接

学者腔长句的三种组织方式：

1. **概念递归**：A 由 B 界定，B 由 C 界定，层层下沉；层数受 `output_length` 约束（M10）。
2. **层级分类**：先立上位范畴，再分下位类型，最后就个别类型处理；分类标准必须单一且明示。
3. **概念套接**：以“作为……的……”结构把概念嵌入上位概念；套接不得超过两级，否则拆句。

```text
递归式：所谓信赖保护，系指对合理信赖之保护；而合理信赖，又以表意人可归责之权利外观为前提。
层级式：权利之限制，大别为内部限制与外部限制；前者系权利本有之边界，后者则为本于公共利益之干涉。
套接式：作为法律行为效力控制机制之诚实信用原则，在给付障碍法中，具体化为附随义务之违反。
```

## 十、段群组织：学者腔连续表达

完整学者腔段落通常循下列链式结构组织（各环可删，删除后命题仍须完整）：

```text
提出问题 → 排除表面理解 → 概念界定 → 体系定位 → 价值证成 → 谨慎收束
```

```text
【问题】之处理，向为学说所争（提出）。表面观之，似可以【日常理解】概之；然此说未见其的（排除）。所谓【概念】，实系【界定】（界定）。置于【上位制度】之中，其功能在于【功能定位】（定位）。究其根本，则涉及【价值或原则】之权衡（证成）。惟此种权衡，仍应受【来源已有之限制】之节制，未可一概而论（收束）。
```

- `rewrite`：只保留“界定”与“收束”两环的等值内容；
- `expand`：每增加一环计入 `expansion_budget`；
- `analyze`：全链可用，但每环须有依据。
