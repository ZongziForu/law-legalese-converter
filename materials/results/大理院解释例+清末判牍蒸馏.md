> 注：由于原 Markdown 带有 OCR 残痕、断行、页码、误字，以下统计以文本机制为目的，不把频次作为严格学术计量。

### 0.2 重新检查后发现上一版不足之处

上一版已经提炼了“主体抽象化、行为名词化、因果链条、体系化、否定重构、条件化表达”等核心机制，但还不够全面，主要缺失如下：

1. **对《各省审判厅判牍》的蒸馏不足**  
   该文本不是单纯法律解释，而是“批词裁断体”。其核心黑话感不在于现代教义学，而在于：  
   - 证据审查式责难；
   - 呈词漏洞揭示；
   - “何以”“岂能”“足征”“显系”的推断式裁断；
   - “碍难照准”“不准”“候行查覆核夺”的衙门式结论；
   - 对缠讼、妄控、狡辩、支离、图赖、讹索等行为的道德化法律评价。

2. **对公牍流程套语提炼不足**  
   两份材料都强烈依赖“准……函开 / 案据…… / 本院查…… / 相应函复 / 合行牌示 / 仰即遵照”等流程语言。这是制造庄严感的重要机制。

3. **对“古典法言法语”和“现代法教义学”的分层不足**  
   有些词可直接用于现代高冷法学文；有些只适合搞怪古风，不宜现代论文使用。需要标注“现代可用 / 仅适合仿古 / 可改造成现代术语”。

4. **对部门法口味不够细分**  
   语料中同时有刑法、民事、诉讼、商事、破产、公司、婚姻、宗法、司法行政等表达。skill 应允许切换口味。

5. **对“结论输出格式”提炼不足**  
   文本中结论很少直接说“可以/不可以”，而是写成：  
   - “自应……”
   - “尚难谓……”
   - “不能构成……”
   - “应仍照原判办理”
   - “碍难照准”
   - “候行查覆再行核夺”
   - “相应函复查照”
   这些应进入结论模板库。

# 一、文本整体风格卡

## 1.1 总体风格判断

| 风格维度 | 判断 | 强度 | 说明 |
|---|---:|---:|---|
| 现代法教义学 | 中强 | 3.5/5 | 《大理院解释例》已有“要件、效力、适用、但书、准用、职权、范围、构成”等教义学雏形。 |
| 古典法言法语 | 极强 | 5/5 | 两份材料大量使用“惟、兹、据、案关、未便、殊、抑、苟、俾、仰、核夺、照章”等。 |
| 德日理论翻译腔 | 中等 | 3/5 | 不是现代德式论文腔，但已有“构成要件—适用范围—效力发生—条件成就—准用”的结构。 |
| 民国/清末法学文风 | 极强 | 5/5 | 兼具旧式公牍、清末律例、新式诉讼、早期法典术语。 |
| 哲学化/体系化 | 中强 | 3.5/5 | 更偏“原则、主旨、本意、制度、范围”，不是纯哲学思辨。 |
| 搞怪黑话适配性 | 极强 | 5/5 | 尤其适合把日常小事转为“案关疑义、要件不合、碍难照准、俾资遵循”。 |

## 1.2 两份语料的不同气质

### A. 《民国大理院解释例全文》的表达气质

核心气质：**统一解释机关的高位、审慎、条文化、体系化表达**。

典型语言引擎：

1. **解释权威感**  
   - 本院查……
   - 统一解释之权……
   - 相应函复……
   - 查照可也……

2. **条文切割感**  
   - 欲援用但书，须先解释前段。
   - 该规定以某范围为限。
   - 不能统括解释为一切情形。

3. **效力时间感**  
   - 法律不溯及既往。
   - 效力发生。
   - 停止条件。
   - 未发生效力前。

4. **要件排除感**  
   - 要件不合，不能构成该罪。
   - 不能谓为……
   - 尚难论以……

5. **程序职权感**  
   - 依职权调查。
   - 是否准许，应由法院裁量。
   - 抗告、再审、管辖、上诉、声明、回复原状等。

适合 skill 的用途：  
**把普通问题提升为“条文解释、要件判断、效力范围、程序权限”的法教义学黑话。**

---

### B. 《各省审判厅判牍》的表达气质

核心气质：**批词裁断、证据责难、古典官箴、民刑交错的裁判文牍表达**。

典型语言引擎：

1. **案情压缩感**  
   - 准某机关移送……
   - 查某人控某人某事……
   - 经某厅讯明……
   - 仍照原判……

2. **证据批驳感**  
   - 何以……
   - 岂能……
   - 一面之词岂能凭信。
   - 证据毫无。
   - 情词种种支离。

3. **道德化裁断感**  
   - 显系畏罪，希图翻案。
   - 殊属好讼。
   - 无理混狡，意在拖累。
   - 妄图翻异。
   - 藉端讹诈。

4. **程序命令感**  
   - 候行查覆核夺。
   - 仰即遵照。
   - 应即驳斥。
   - 不准。
   - 合行牌示。

5. **情理折中感**  
   - 酌理准情。
   - 尚属允协。
   - 姑候……
   - 情尚可原。
   - 以息讼端。

适合 skill 的用途：  
**把普通争吵改造成“官府批词式、证据审查式、责难式、古风裁判式黑话”。**

# 二、语言复杂化机制

## 2.1 主体抽象化

### 机制说明

普通表达中直接出现“我、你、他、老板、室友、朋友、商家、平台”等具体生活主体。法言化时，将其替换为处于法律关系中的抽象角色。

### 替换表

| 普通主体 | 法言法语替换 |
|---|---|
| 我 | 请求权人、原告、呈控人、上诉人、权利主体、受损害一方 |
| 你 / 他 | 相对人、被告、行为人、义务主体、被控人、对造 |
| 两个人 | 两造、双方当事人、甲乙、争讼双方 |
| 帮忙的人 | 关系人、从行为人、辅助人、见证人、凭中 |
| 老板 | 用工主体、管理主体、给付义务人、组织支配者 |
| 商家 | 经营主体、交易相对方、出卖人、服务提供者 |
| 平台 | 平台经营者、信息媒介主体、管理义务主体 |
| 室友 | 共同生活关系之一方、共同空间使用人 |
| 群主 | 管理权限主体、秩序维持主体 |
| 法院/机关 | 审判机关、该管机关、原审衙门、上诉衙门、本院、本厅 |

### 可复用模板

```text
【某人】不宜仅以自然生活中之个人理解，而应视为处于【法律关系】中，享有【利益】或负有【义务】之主体。
```

```text
本案所涉者，并非【甲乙】之间普通往还，而是【权利主体】与【义务主体】之间因【事实】所形成之规范关系。
```

```text
判断【行为】是否可责，不能仅从行为人主观好恶出发，而应自其在【制度关系】中所处之主体地位加以考察。
```

## 2.2 案由化：把生活小事包装成“案关疑义”

### 机制说明

日常语言直接说事情；法言法语先给事情套上“案由、争点、疑义、请求、声明、抗辩”。

### 转换表

| 大白话 | 案由化表达 |
|---|---|
| 他欠钱不还 | 关于金钱给付义务不履行之争议 |
| 他骗了我 | 关于欺罔陈述介入意思形成之争议 |
| 室友不洗碗 | 关于共同生活秩序维持义务怠于履行之争议 |
| 老板不给工资 | 关于劳动报酬给付迟延之争议 |
| 群里骂我 | 关于群体交流空间中人格利益减损之争议 |
| 外卖迟到 | 关于履行迟延及时间利益减损之争议 |
| 快递丢了 | 关于运输保管过程中占有支配状态灭失之争议 |

### 可复用模板

```text
本案争点，非在【事实】是否令人不快，而在【事实】是否足以构成【规范争议】。
```

```text
就【普通事情】而言，其法律上应表述为关于【权利/义务/状态】之争议。
```

## 2.3 行为名词化

### 机制说明

这是最重要的黑话引擎。普通动词必须变成抽象名词，再连接“义务、状态、效果、评价”。

### 高频转换方式

| 普通动词 | 名词化黑话 |
|---|---|
| 骗 | 欺罔、欺罔性陈述、错误认识之诱发、意思形成过程之介入 |
| 打 | 殴击行为、身体完整性侵害、物理力作用 |
| 欠 | 给付义务不履行、清偿迟延、债务不履行 |
| 拿走 | 占有状态转移、支配关系变动、财物控制脱离 |
| 不来 | 到场义务不履行、程序协力义务懈怠 |
| 不说 | 告知义务违反、信息披露不足 |
| 乱说 | 不实陈述、虚伪陈述、名誉评价减损行为 |
| 拖 | 履行迟延、程序迟滞、延不办理 |
| 搞混 | 标的混同、事实关系不明、权属状态含混 |
| 闹 | 争讼、纷争、缠讼、扰乱秩序 |
| 不认账 | 抗辩、否认、拒绝履行、狡供不认 |
| 找借口 | 藉词推诿、托词规避、藉端抗拒 |

### 可复用模板

```text
【动词】 → 对【法益/利益/状态】之【侵害性/变动性/妨害性】作用。
```

```text
【不做某事】 → 对【义务】之怠于履行，致【利益状态】处于未实现或受阻状态。
```

```text
【说了某话】 → 通过【信息输入】介入相对人之【认识状态】与【意思形成过程】。
```

## 2.4 因果链条复杂化

### 机制说明

不要说“因为 A 所以 B”。要拆成若干中间环节：行为、认识、意思、处分、状态、损害、归责。

### 常见链条

#### 欺骗类链条

```text
欺罔性信息输入
→ 相对人认识状态发生偏差
→ 意思形成过程受介入
→ 作出财产处分或行为选择
→ 财产利益或机会利益减损
→ 损害与行为之间形成规范归责关联
```

#### 打人类链条

```text
物理力作用
→ 身体完整性受影响
→ 健康利益或人格安宁发生减损
→ 损害后果可被证据确认
→ 进入侵权、治安或刑事评价
```

#### 欠钱类链条

```text
债务成立
→ 履行期届至
→ 给付义务未被实现
→ 债权人财产期待落空
→ 清偿迟延或履行不能
→ 责任范围确定
```

#### 程序类链条

```text
申请/声明提出
→ 程序入口审查
→ 期间、管辖、形式、主体资格判断
→ 合法则进入实体审查
→ 不合法则决定驳回或不予准许
```

### 可复用模板

```text
【行为】并非直接产生【结论】，而是经由【中间状态一】、【中间状态二】及【法律效果】之连续转化，始进入【责任/效力/程序】评价。
```

## 2.5 条文切割与但书解释

### 机制说明

《大理院解释例》最强的机制之一，是把一个条文拆成“前段、后段、但书、例外、范围、明示、衍文、立法本意”。

### 操作步骤

1. 先说不能轻易看过条文。  
2. 再说欲援用但书，须先解释前段。  
3. 再确定原则。  
4. 再限定例外。  
5. 最后以立法主旨或条文本意收束。

### 可复用模板

```text
欲援用【例外/但书】之规定，要先解释【前段/原则】之规定；若不先明其前段文义，而径行援用例外，则不免扩大适用范围，殊非制度本意。
```

```text
【规则】既系对于【一般原则】之例外，自应从严解释，并以条文明示者为限。
```

## 2.6 效力时间化

### 机制说明

把“什么时候算数”表达成“效力发生、效力范围、溯及力、停止条件、确定、未确定”。

### 常见表达

- 效力发生
- 效力不溯既往
- 施行以前
- 公布以后、发生效力以前
- 确定审判前
- 条件成就
- 停止条件
- 已确定 / 未确定
- 自某时起算

### 可复用模板

```text
该规则之效力并非当然及于一切既存事实，其是否适用，尚须区分【发生时间】、【生效时间】与【法律效果完成时间】。
```

```text
若【事实】发生于【规则】效力发生之前，除条文特别明示外，不得当然溯及适用。
```

## 2.7 要件化与构成化

### 机制说明

普通判断“算不算”变成“是否具备要件”。这是现代法学黑话最稳的入口。

### 可复用结构

```text
该行为是否构成【评价】，应审查：
一、主体是否适格；
二、客体是否存在；
三、行为是否具备外部表现；
四、结果是否发生；
五、行为与结果之间是否具有规范关联；
六、有无阻却或例外事由。
```

### 典型模板

```text
仅有【事实】尚不足以构成【法律评价】，仍须具备【要件一】、【要件二】及【要件三】，始能成立。
```

```text
【行为】虽有不当，然与【规则】所要求之要件尚有距离，故未可径谓构成【责任】。
```

## 2.8 程序化与权限化

### 机制说明

把实体问题转成程序问题，把“谁对谁错”转成“谁有权判断、按什么程序判断”。

### 转换表

| 普通说法 | 程序化表达 |
|---|---|
| 这事该怎么办 | 应由何机关依何程序处理 |
| 你说了不算 | 非该主体所得径行决定 |
| 要不要管 | 是否进入受理范围 |
| 要不要重新来 | 是否准许再审、抗告或回复原状 |
| 现在还能不能说 | 是否逾越期间 |
| 该不该听他说 | 是否具备程序参与资格 |

### 可复用模板

```text
此项问题，先不问实体上是否有理，尚须先行审查其是否符合【管辖、期间、形式、主体资格】等程序入口条件。
```

```text
【主体】是否得为该项处置，非其主观意愿所能决定，尚须视其是否享有相应之职权基础。
```

## 2.9 证据门槛化

### 机制说明

《判牍》的强项是证据批驳。它不直接说“你瞎说”，而是说“证据毫无、一面之词、情词支离、无从悬揣、岂能凭信”。

### 证据批驳词组

- 一面之词
- 不能托诸空言
- 无确实证据
- 毫无证佐
- 无从悬揣
- 难以凭信
- 尚未讯明
- 虚实均应澈究
- 证据确凿
- 众证确凿
- 据以认定
- 足资认定

### 可复用模板

```text
仅据【一方陈述】，尚难谓【事实】已达可资认定之程度；若无【证据】相互印证，未便径予采信。
```

```text
【主张】若属真实，何以【反常事实】？其情词前后支离，殊难深信。
```

```text
案关【重大利益】，虚实均应澈究，不得仅凭空言遽作断定。
```

## 2.10 “何以—岂能—足征”推断式责难

### 机制说明

《判牍》常通过连续反问制造裁判者的压迫感。

### 句法

```text
若【事实】果真如此，何以【反常行为】？
```

```text
仅有【薄弱根据】，岂能作为【重大结论】之标准？
```

```text
【行为】前后不符，足征其【主观评价】。
```

### 可复用示例

```text
该室友若果有履行之意，何以屡经催告仍置若罔闻？
```

```text
仅凭一句“我忘了”，岂能排除其怠于履行共同生活义务之事实？
```

```text

### 模板 1：前段—但书模板

```text
欲援用【但书/例外】之规定，须先解释【前段/原则】之文义；若不先明其前段范围，而径行扩张例外，殊非【制度/条文】之本意。
```

适合部门法：全领域，尤其条文解释  
晦涩程度：5  
示例：  
```text
欲援用“临时有事”之例外，须先解释轮值义务之前段文义；若不先明其适用范围，而径行扩张免责，殊非宿舍自治规则之本意。
```

### 模板 2：效力发生模板

```text
【规则/约定】虽已成立，惟其效力之发生，尚须以【条件/期限】成就为准；该条件未成就以前，不能谓【效果】当然发生。
```

适合部门法：民法、程序法  
晦涩程度：5

### 模板 3：例外从严模板

```text
【规则】既系对于【一般原则】之例外，自应以明文所示者为限，不得统括解释为一切【类似事实】。
```

适合部门法：全领域  
晦涩程度：5

### 模板 4：要件不合模板

```text
【行为】虽与【某规则】所欲规制之事实近似，惟其【关键要素】尚未具备，自与该规则之要件不合，不能构成【评价】。
```

适合部门法：刑法、侵权、违约  
晦涩程度：5

### 模板 5：程序入口模板

```text
【请求】是否有理，尚非首先应问之事；其能否进入实体审查，仍须先就【主体资格、期间、管辖、形式】等程序入口条件加以判断。
```

适合部门法：诉讼法  
晦涩程度：4

## 4.2 判牍批词式模板

### 模板 6：查核—证据—结论模板

```text
查【事实】，既据【证据】足资认定，则【主张】显系【评价】，应即【处理结果】；惟【例外情节】，尚应【补充处理】。

晦涩程度：5

### 模板 7：何以责难模板

```

适合部门法：证据批驳  
晦涩程度：5

### 模板 8：一面之词模板

适合部门法：证据法、古风裁断  
晦涩程度：5

### 模板 9：缠讼斥断模板

```

适合部门法：批词、搞怪文本  
晦涩程度：5

```text

```

## 6.2 伪代码式规则

```text
若输入包含“他/你/我”：
    替换为“行为人/相对人/权利主体/义务主体”。

若输入包含普通动词：

```

## 示例 2：室友不洗碗

大白话：

```text
室友不洗碗。
```

法言法语黑话：

```text
共同生活关系之一方，于轮值义务已经发生且经相对人催告后，仍怠于履行共用器具清洁及生活秩序维持义务，致共同空间卫生状态发生不利益变动。其行为表面上为洗涤行为之欠缺，实质上则为共同生活共同体内部权义配置失衡之表现。

```text
老板拖工资。

则自应依法进入【责任/救济/处理】程序。”

# 十二、报告结论

这批语料的最大价值，不在于具体法律知识，而在于提供了三套可叠加的黑话生成引擎：

1. **解释例引擎**：  
   把问题变成“条文文义、但书范围、效力发生、要件是否具备”的法教义学问题。

2. **判牍批词引擎**：  
   把问题变成“证据是否足、情词是否支离、何以岂能、应否照准”的裁断问题。

3. **公牍庄严引擎**：  
   把问题变成“案关疑义、未便擅拟、俾资遵循、相应函复、合行牌示”的权威文书问题。

用于 skill 时，推荐默认采用：

```text
40% 解释例条文腔
30% 判牍批词腔
20% 现代教义学抽象腔
10% 仿古公牍搞怪腔
```

这样最能形成“既像法学论文、又像旧式判牍、还带一点不说人话的庄严荒谬感”的稳定输出风格。
