# 转换机制与模式门控

> 机制是候选工具，不是必须全部执行的步骤。模式权限、篇幅预算与命题保真优先于机制偏好。

## 二、基础机制

<!-- chunk: r02.h003 -->
### M01 主体功能化

把生活身份替换为由原关系能够确定的功能称谓。

- “平台”可写为“信息处理主体”或“交易秩序组织者”，前提是文本确实涉及相应功能；
- “对方”可写为“相对人”；
- 不得把“室友”升级为负有法律义务的“义务主体”，除非原文已有义务命题。

<!-- chunk: r02.h004 -->
### M02 行为名词化

把动词写成过程、作用或关系变动：

```text
骗 → 欺罔性信息输入／欺罔行为
给钱 → 财产处分／财产性给付
打人 → 对身体完整性的物理侵害
不回复 → 回应行为的缺位
交货 → 给付行为的现实履行
```

只转换行为性质，不新增秘密性、暴力程度、持续时间、动机或结果。

<!-- chunk: r02.h005 -->
### M03 客体与关系抽象化

```text
钱 → 财产利益／金钱给付
钱包 → 特定动产／具体物
信息 → 数据或信息利益
合同 → 合意形成的法律关系载体
```

严格区分：

- 日常“他的东西”不自动等于法律所有权；
- “拿走”不自动等于所有权转移；
- “控制”不自动等于占有；
- “主张”不自动等于事实已查明。

<!-- chunk: r02.h006 -->
### M04 明示结构链条化

仅在来源已经包含各环节时展开：

```text
行为 → 中间状态 → 后续行为 → 结果
```

例如原文已写明“欺骗—认识错误—处分”，可显化其连续关系；若只写“骗钱”，不得自动补齐全部诈骗构造。

<!-- chunk: r02.h007 -->
### M05 否定—重构

把简单表达改为“并非仅是……而是……”等结构，但两端必须是同一命题的不同抽象层次，不得凭空制造错误观点或学说对立。

<!-- chunk: r02.h008 -->
### M06 条件、例外与情态显化

- 来源有“只有……才……”时，可展开条件门槛；
- 来源有“原则上／除非”时，可展开原则—例外；
- 来源只有“可能”时，所有改写都必须保留可能性；
- 不得为了句式完整而自造条件、例外或阻却事由。

<!-- chunk: r02.h009 -->
## 三、expand 专属机制

<!-- chunk: r02.h010 -->
### M07 体系位置标注

可把原现象置于上位结构中，例如：

- 意思形成；
- 信赖保护；
- 风险分配；
- 权利边界；
- 程序正当；
- 制度功能。

在 `rewrite` 中只能使用与原词等值的通用分类，不得引入新的理论视角。

<!-- chunk: r02.h011 -->
### M08 概念区分与类型化

可增加一个与原问题直接相关的区分，例如：

- 事实状态与法律效果；
- 义务存在与义务履行；
- 主张提出与主张成立；
- 形式控制与实质控制。

区分用于整理表达，不得借机作出原文没有的结论。

<!-- chunk: r02.h012 -->
### M09 制度功能与价值框架

可从交易安全、意思自治、信赖保护、法益保护、程序参与、风险分配等角度说明问题可能涉及的功能。必须使用非断言式表达，不得宣称某一价值已经压倒另一价值。

<!-- chunk: r02.h013 -->
### M10 同义定义递归

```text
所谓A，系指B；换言之，是C；从关系结构看，又可表述为D。
```

- `rewrite`：B、C、D 必须严格同义；
- `expand`：可增加一个上位框架，但计入 `expansion_budget`；
- 递归层数受篇幅约束：minimal 0，short 1，standard 2，long 3，extended 4。

<!-- chunk: r02.h014 -->
### M11 外国概念或比较视角

- 只使用可靠、通行的术语；
- `expand` 中只能作为观察框架，不得推定现实法适用；
- 每段默认不超过一个外语术语；
- `authority_policy=none` 时不得附会学者、国家法、页码或“通说”。

<!-- chunk: r02.h015 -->
## 四、analyze 专属机制

<!-- chunk: r02.h016 -->
### M12 阶层检验与请求权基础

三阶层、请求权基础、客观归责、构成要件涵摄等完整检验结构仅在 `analyze` 使用。

`argumentative_overkill=5` 也不能在 `rewrite` 或 `expand` 解锁本机制。

<!-- chunk: r02.h017 -->
### M13 学说对立与权威引用

只有在以下条件之一成立时使用：

- 用户提供了具体材料；
- 工具已经核验；
- 仅以无归属的一般概念分歧作条件性说明，且不伪称“通说”。

不得用空白占位符生成“某学者、某国法、某判例”。

<!-- chunk: r02.h018 -->
## 五、语体与搞怪机制

<!-- chunk: r02.h019 -->
### M14 语体滤镜

语体滤镜只改变词面、节奏和句法：

- 老派民法：半文半白、概念分层、审慎限定；
- 当代裁判：事实／主张／争点层次清楚；
- 古典法言：按、盖、惟、是以、未可遽谓；
- 民国判牍：仅在明确历史请求时开放时代性格式。

“翌日、旋、嗣后、迨至”等时间词，以及“显有、綦详”等程度词，只有来源存在相应时间或程度信息时才能使用。

<!-- chunk: r02.h020 -->
### M15 过度论证与庄严反差

搞怪可以来自：

- 对微小事项使用宏大的规范语言；
- 对同一命题进行多层同义界定；
- 使用严肃、审慎、体系化的句法描述日常行为。

不得来自：

- 虚构理论或外语；
- 编造持续时间、频率、劳动耗时或社会后果；
- 以新增法律责任制造笑点；
- 无视 `output_length` 无限递归。

<!-- chunk: r02.h021 -->
### M16 长文本一致性

建立内部术语表，保留原格式；对主体、对象、时间、认识状态和法律关系作跨段一致性检查。每段最多一个高漂移机制。

## 六、机制选择预算

| 文本 | 默认机制数 |
|---|---:|
| 单句 | 2—4 |
| 短段 | 3—5 |
| 长段／全文 | 每段2—4 |

高风险机制（M08—M13）每段默认最多一个；`expansion_budget` 与 `output_length` 可能进一步降低数量。

## 七、句群与段落组织机制

下列机制影响整句话、整个句群的组织方式，而不只是替换单词。各机制的可用范围仍受任务模式门控：涉及新概念、新框架者仅限 `expand`/`analyze`。

| ID | 句群机制 | rewrite | expand | analyze |
|---|---|---:|---:|---:|
| P01 | 问题提出—表面理解排除 | 是（来源已有排除对象时） | 是 | 是 |
| P02 | 事实描述—规范评价提升 | 仅来源已有评价 | 是 | 是 |
| P03 | 构成要件—体系定位 | 否 | 是 | 是 |
| P04 | 二说区分—谨慎选择 | 否（来源须已有二说） | 是 | 是 |
| P05 | 原则—例外—例外之限制 | 仅来源已有 | 是 | 是 |
| P06 | 概念源流—制度功能—保护目的三元论证 | 否 | 是 | 是 |
| P07 | 递归定义—层级分类—概念套接 | 有限（同义递归） | 是 | 是 |
| P08 | 古典式段落推进（按—盖—惟—然则—准此以观） | 是（词面） | 是 | 是 |

<!-- chunk: r02.h024 -->
### P01 问题提出—表面理解排除

```text
就【事项】而言，表面观之似为【日常理解】；然细究之，其所涉者实为【等值的抽象表述】。
```

排除对象必须是来源中真实存在的日常理解，不得虚构一个“通常见解”作为靶子。

<!-- chunk: r02.h025 -->
### P02 事实描述—规范评价提升

```text
【事实层描述】；进入规范评价层面，同一事实即呈现为【规范层表述】。
```

`rewrite` 中规范层表述必须与事实层等值；`expand` 中方可增加评价框架。

<!-- chunk: r02.h026 -->
### P03 构成要件—体系定位

```text
就个别要件而言，关键在于【要件分析】；就体系位置而言，该要件又处于【上位制度】之脉络中。
```

仅限 `expand`/`analyze`；`rewrite` 不得补造体系位置。

<!-- chunk: r02.h027 -->
### P04 二说区分—谨慎选择

```text
关于【问题】，向有【甲说】与【乙说】之分；二说各有所据，本文暂从【保持来源立场或悬置】。
```

来源无二说时不得虚构对立学说；`authority_policy=none` 时不得冠以“通说”。

<!-- chunk: r02.h028 -->
### P05 原则—例外—例外之限制

```text
【原则】固应如此；惟【例外情形】不在此限；而例外之适用，仍以【限制条件】为界。
```

三层结构必须逐层有来源依据；来源只有原则时，只写原则层。

<!-- chunk: r02.h029 -->
### P06 概念源流—制度功能—保护目的三元论证

```text
【概念】之设，原以【源流或文义】为据；就制度功能言，其在【功能定位】；就保护目的言，则指向【所保护之利益或关系】。
```

`expand` 中每增加一元通常计入一个 `expansion_budget` 单位。

<!-- chunk: r02.h030 -->
### P07 递归定义—层级分类—概念套接（长句组织）

```text
所谓【概念A】，系指【界定】；而【界定中的概念B】，又可区分为【子类一】与【子类二】，其中【子类一】复以【下位标准】为断。
```

递归层数服从 `output_length` 上限（见 M10）；每层嵌套必须保持可回译。

<!-- chunk: r02.h031 -->
### P08 古典式段落推进

```text
按：【叙事】。盖【原理】，惟【限制】。然则【推论】。准此以观，【收束】。
```

推进词不得堆砌：一段之内“盖”“惟”各至多一处；“准此以观”仅在确有前文依据时收束。

## 八、表达强度梯度

同一意思可保留数个强度不同的表达，供 `intensity` 档位调用；梯度只影响表达强度，不改变三种任务模式的内容权限。

以“他不还钱”为例：

| 档位 | 表达 | 特征 |
|---|---|---|
| 1 轻度法学化 | 其未履行到期金钱债务 | 术语替换，短句 |
| 2 中度教义学化 | 债务人于清偿期届满后，仍未为金钱给付之履行 | 名词化＋时间结构 |
| 3 高度抽象化 | 该金钱之债既届清偿期而未获现实履行，债之拘束力即于时间维度上持续发生作用 | 关系重构＋体系定位 |
| 4 荒诞过度化（搞怪） | 所谓届期不清偿，并非仅系货币占有状态之持续未变更，而是债务关系拘束力在时间轴上展开过程中，现实履行环节之结构性缺位 | 多层同义递归、庄严反差 |

梯度标注义务：输出中不得混用相邻两档以上而不自觉——全文强度浮动不超过一级（见 SKILL.md 长文本一致性）。

## 九、词族归并规则

“某某义务”“某某效力”“某某功能”类词族可以无限机械衍生，禁止以填充体积为目的录入。归并规则：

1. **词族只收原型与不可推导成员**：收“附随义务”，不逐条收“通知义务、保密义务、照顾义务”——后者由“附随义务＋来源明示的具体内容”生成；
2. **可推导成员仅在以下情形单列**：(a) 有独立的文体变体或历史语体形式；(b) 有独立的误用风险；(c) 跨部门法含义不同；
3. **强度标注以词族为单位**：词族内成员共享强度档，不因字数增减而变化；
4. 词族调用时，具体内容必须来自来源文本，不得为了词族完整而补造义务清单。

## 十、因果双链（M04 补充）

复杂化表达中存在两条应区分的链：

```text
事实链：行为 → 自然事实上的中间状态 → 结果
规范链：行为 → 规范评价上的归责节点 → 法律效果
```

以诈骗结构为例：事实链为“欺罔表示—错误认识—财产处分—财产变动”；规范链为“欺罔行为之不法—处分行为与错误之关联—财产损害之归属”。

规则：

1. `rewrite` 只能显化来源已有的链环节，不得补齐缺失环节；
2. 事实链与规范链不得混接——“导致损失”（事实链）不等于“损害应予归属”（规范链）；
3. 规范链的完整展开（如客观归责检验）仅限 `analyze`；
4. 双链并用时先事实后规范，不得倒叙造成结论先行。
